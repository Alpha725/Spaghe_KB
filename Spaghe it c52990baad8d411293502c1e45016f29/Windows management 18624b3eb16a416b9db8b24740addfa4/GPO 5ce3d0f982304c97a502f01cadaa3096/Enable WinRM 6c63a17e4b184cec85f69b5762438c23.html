Enable WinRM

First, we need to create a Group Policy object for your domain.

  1. From the start menu, open Control Panel.
  1. Select Administrative Tools.
  1. Select Group Policy Management.
  1. From the menu tree, click Domains > [your domain’s name].
  1. Right-click and select Create a GPO in this domain, and Link it here.
  1. Input Enable WinRM.
  1. Click OK.

Next, edit the new Group Policy object you just created. When you’re done, there will be three WinRM service settings enabled:

  • Allow remote server management through WinRM
  1. Right-click on the new Enable WinRM Group Policy Object and select Edit.
  1. From the menu tree, click Computer Configuration > Policies > Administrative Templates: Policy definitions > Windows Components > Windows Remote Management (WinRM) > WinRM Service.
  1. Right-click on Allow remote server management through WinRM and click Edit.
  1. Select Enabled to allow remote server management through WinRM.
  1. Enter an asterisk (*) into each field.
  1. Click OK.

Now that Windows Remote Management has been enabled on the Group Policy, you need to enable the service that goes with it.

  1. From the Group Policy Management Editor window, click Preferences > Control Panel Settings > Services.
  1. Right-click on Services and select New > Service.
  1. Select Automatic as the startup.
  1. Enter WinRM as the service name.
  1. Select Start service as the service action.
  1. All remaining details can stay on the defaults. Click OK.

Now you must allow for inbound remote administration by updating the firewall rules. When you’re done, there will be two rules enabled:

  • Windows Firewall: Allow inbound remote administration exception
  • Windows Firewall: Allow ICMP exception
  1. Using the Group Policy Management Editor, from the menu tree, click Computer Configuration > Policies > Administrative Templates: Policy definitions > Network > Network Connections > Windows Firewall > Domain Profile.
  1. Right-click on Windows Firewall: Allow inbound remote administration exception and click Edit.
  1. Select Enabled.
  1. Enter the IP address into the field called Allow unsolicited incoming messages from these IP addresses. Enter the range that you want to be able to access
  1. Click OK.
  1. Right-click on Windows Firewall: Allow ICMP exception and click Edit.
  1. Select Enabled.
  1. Check Allow inbound echo request.
  1. Click OK.

Almost done! The final steps is to create a new inbound firewall rule and update the network list manager for unidentified networks.

  1. From the menu tree, click Computer Configuration > Policies > Windows Settings > Security Settings > Windows Firewall with Advanced Security > Windows Firewall with Advanced Security > Inbound Rules.
  1. Right-click on Inbound Rules and click New Rule.
  1. Select Predefined.
  1. Select Windows Remote Management from the list of services.
  1. Click Next.
  1. Uncheck the Public rule. Leave the Domain, Private rule checked.
  1. Click Next.
  1. Leaving the defaults, click Finish.
  1. Right-click on the new rule and click Properties.
  1. Click the Advanced tab.
  1. Uncheck Private.
  1. Click OK.
  1. From the menu tree, click Computer Configuration > Policies > Windows Settings > Security Settings > Network List Manager Policies.
  1. Right-click Unidentified Networks and click Properties.
  1. Change the location type from Not configured to Private.
  1. Click OK.
  1. Close the Local Group Policy Editor window.
