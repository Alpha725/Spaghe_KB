Managing containers

Overview

  • Find and retrieve container images from a remote registory
  • Inspect containers
  • Perform container management using commands such as:
    • podman
    • skepeo

Commands:

  • Podman
    • Used to directly manage containers and container images
  • skopeo
    • used to inspect copy delete and sign container images
  • buildah
    • used to create new container images

Container registries:

  • registory.redhat.io (RedHat internal)
  • registory.connect.redhat.com (RedHat community images)

Configuration: /etc/containers/regitories.conf

Example: 

## lists avaliable modules and versions
sudo yum -y module list --avaliable container-tools
## installed container-tools module
sudo yum -y module install container-tools
podman --help | less
podman info | less
## logs into the registry for RHEL images
podman login registry.access.redhat.com
## looks for universal basic image for RHEL docker image and list all avaliable
podman search registry.access.redhat.com/ubi8 | less
## pulls the image
podman pull registry.access.redhat.com/ubi8/ubi:latest
## lists all the images
podman images
## Gives you information on the image in a json format
skopeo inspect docker://registry.access.redhat.com/ubi8/ubi:latest | less
## starts a container and then drops you into an interactive shell
podman run -it registry.access.redhat.com/ubi8/ubi:latest
	cat /etc/redhat-release
	exit
## List all docker containers, running or otherwise
podman ps --all 
## removes all docker images from a machine
podman rm -a 
## removes all images on the machine
podman rmi registry.access.redhat.com/ubi8/ubi:latest

Basic Operations

  • find and retrieve container images from a remote registry
  • inspect container images
  • Perform basic container management such as running, starting, stopping, and listing running containers
  • Perform container management using commands such as podman and skopeo
  • Run a service inside a container
  • Attach persistent storage to a container

Example: 

## inspects image, you do not need to pull it first ;)
skopeo inspect docker://registry.access.redhat.com/rhscl/httpd-24-rhel7 | less 
## starts docker container using the httpd-24-rhel7 image
## maps port 8080 to 8080 on the container
podman run -d -p 8000:8000 docker://registry.access.redhat.com/rhscl/httpd-24-rhel7
## Lists all containers
podman ps -a
## testing the above
curl http://127.0.0.1:8000
## stopping all containers
podman stop -a
## lists all containers
podman ps -a
## starts a container and then drops you into an interactive  bash shell
podman run -it registry.access.redhat.com/rhscl/httpd-24-rhel7 /bin/bash
	cat /etc/rhel-release
	exit
## starts a container 
podman run -d --name test-container  docker:registry.access.redhat.com/rhscl/httpd-24-rhel7
## executes command against container 
podman exec -it test-container cat /etc/rhel-release
## stops the container
podman stop test-container
## creating a container and mapping a local dir to /var/www/html and maps ports 8000
mkdir ~/web_data
echo "Test data" > ~/web_data/test.txt
cat "Test data" > ~/web_data/test.txt
podman run -d --name web_container -p 8000:8000 -v ~/web_data:/var/www/html:Z registry.access.redhat.com/rhscl/httpd-24-rhel7
podman ps -a 
curl http://127.0.0.1:8000/test.txt
## list all, stop all, remove all
podman ps -a 
podman stop -a 
podman rm -a

Managing containers on RHEL 8 using podman - Advanced operations find and retrieve container images from a remote registry perform container management using commands such as podman and skopeo perform basic container management such as running , starting, stopping and listing running containers Run a service inside of a container Configure a container to start automatically as a systemd service Attach persistent storage to a container

Commands: podman run -d -name Launch a new container to use as a model to generate the systemd unit files podman generate systemd Generate systemd unit file from your container. Must delete the container as systemd will create one new one podman stop|rm Get rid of the container you created before you try to start your systemd container systemd commands: loginctl enable-linger loginctl disable-linger loginctl show-user systemctl –user daemon-reload systemctl –user start|stop|enable UNIT Unit files: ~/.config/systemd/user/

Examples: 

## setting up folder to generate a docker image
mkdir -p ~/.conf/systemd/user 
ll ~/.conf/systemd/user 
cd ~/.conf/systemd/user 
mkdir ~/web_data 
echo “test data” > ~/web_data/test.txt 
cat ~/web_data/test.txt 
## starting a container using local dirs and ports
podman run -d –name web_server -p 8000:8000 -v ~/web_data:/var/www/html:Z registry.access.redhat.com/rhscl/httpd-24-rhel7 
## testing
curl http://127.0.0.1/test.txt 
## creating a reusable image to run as a service
podman generate systemd –name web_server –files –new 
## listing the docker images details
less container-web_server.service 
## clean up
podman stop web_server 
podman rm web_server 
## enabling linger on local account, then reloading the daemon
loginctl enable-linger 
loginctl show-user cloud_user | grep linger 
systemctl –user daemon-reload 
## starting the container from the nearly created file
systemctl –user enable –now container-web_server.service 
systemctl –user status container-web_server.service 
## testing it worked
curl http://127.0.0.1/test.txt 
reboot 
## checking persistence across reboots
systemctl –user status container-web_server.service 
curl http://127.0.0.1/test.txt
